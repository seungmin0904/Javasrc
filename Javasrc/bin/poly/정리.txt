인터페이스
- 추상 클래스 ( 단 추상화 정도가 높다 )
- 일반메서드, 멤버변수 가질 수 없음
- 상수, 추상메서드만 가짐
- 부모 역할(자손클래스에서 추상메서드를 반드시 구현하도록 강요)
- 인터페이스도 상속을 받을 수 있다 - 인터페이스로만 상속 가능 - 다중 상속 가능
- 장점: 표준화가 가능하다. / 서로 관계없는 클래스들에게 관계를 맺어 줄 수 있다.

단일 상속
클래스는 단일상속
class A extends B {}


추상 클래스
-  abstract 키워드 사용
- 추상 메서드가 있어야 함
- 인스턴스 생성불가(new 불가)
- 부모 역할 개념으로 사용(자손 클래스에서 추상 메서드를 반드시 구현하도록 강요)
- 추상화 : 클래스간의 공통점을 찾아내서 공통의 조상을 만드는 작업
- 구체화 : 상속을 통해 클래스를 구현, 확장하는 작업



다형성(polymorphism)
- 상속과 관련이 있음
- 여러가지 형태를 가질 수 있는 능력
- 한 타입의 참조변수로 여러타입의 객체를 참조한다.
- 조상 타입의 참조 변수로 자손 클래스의 인스턴스를 참조한다.
- 왼쪽에 선언 된 참조변수의 자식 클래스 메소드가 오버라이딩 됐을 경우 
  부모 쪽이 아닌 자식 클래스 메소드가 호출된다
  - 참조 변수의 타입에 따라 사용 할 수 있는 멤버의 개수가 달라진다
    (부모에 선언 된 멤버만 접근 가능 - 단 오버라이딩 된 메소드가 있다면 
    자식클래스의 메소드가 호출된다)
